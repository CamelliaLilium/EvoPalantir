## 写作顺序（四步）

1. **锚定** — 从 [现实对齐方案](../../knowledge/基于%20EvoPalantir%20的现实对齐模拟方案.md) 抄准 **数据流一句** + **校正记忆职责一句**；列出引用，不扩写。
2. **调研补全** — 在 [自进化Agent…](./自进化Agent：经验写回的运行时记忆闭环机制/自进化Agent：经验写回的运行时记忆闭环机制.md) 上 **只加**：各框架 **→ 本场景一句**；另 **MemOS / OpenViking / Mem0（或等价）** 各 **可核实** 一段，对照 MLflow 留痕。
3. **设计** — **一张图或一列表** 写清模块：**记录 | 索引 | 检索 | 下游（推演 / 人读）**；每块 **输入/输出/不负责什么**；**缺口单列**。
4. **收口** — 合规与数据形态 **选项+风险**；演进 **短段**（不预先排除深度集成）；未决问题 **条列**。

---

## 正文建议结构（五块，无附录仪式）

| 块 | 写什么 |
|----|--------|
| **1. 问题与边界** | 要解决什么、不解决什么；链自需求说明 + 知识库 §4.10。 |
| **2. 数据流与解耦模块** | 快照→校正→记忆→推演；模块间 **只通过约定数据结构/事件** 连接。 |
| **3. 调研摘录** | 框架与开源参照 **短表**；每行 **本场景怎么用**。 |
| **4. 设计方案** | 上表四列能力 + 契约要点 + 缺口。 |
| **5. 合规与未决** | 数据选项；开放问题。 |
